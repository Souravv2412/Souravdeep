Situation
A 48-person professional services firm was losing 14 hours every week to document chaos. Files lived on personal desktops, personal cloud folders, and scattered messaging threads. Version conflicts were routine. Client-facing proposals circulated in three competing drafts at the same time. Teams spent more time tracking down documents than doing actual work.
Task and Objective
Design and implement a clean document governance system on Microsoft OneDrive. The goal was simple: any employee, on any day, should be able to find any document within 60 seconds without asking anyone for help.
Action
What I Built and How
1
Designed a folder structure using strict numerical prefixing (01_Operations, 02_Finance, 03_Client_Services) to prevent buried, hard-to-navigate hierarchies.
2
Established a mandatory file naming convention: YYYYMMDD_DocumentName_vX.X to eliminate version confusion at the file-system level.
3
Built a Star Schema architecture in Excel with a central Master Document Registry (Fact Table) tracking every file by Document ID, version, department owner, and compliance status.
4
Created three linked Dimension Tables: Access Permission Matrix, Regulatory Retention Schedule, and Department Code Directory.
5
Wrote five Standard Operating Procedures covering the full document lifecycle: creation, version control, archiving, new-hire provisioning, and emergency retrieval.
Process Diagram
Folder structure with role-based access permissions across all departments
Result
The Outcome
File-search time dropped from 14 hours per week to under 2 hours company-wide. New hires reached operational independence 35% faster due to the self-service SOP repository. During the migration audit, three expiring high-value vendor contracts were identified and flagged, preventing $4,200 in auto-renewal penalties.
